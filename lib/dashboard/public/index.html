Фильтр вакансий

Общие настройки с вкладкой «Сбор вакансий» (preferences.json). Без пункта «Условие генерации» — на этой вкладке действует один сценарий (см. ниже).

Режим: только вакансии со статусом «На проверке», без черновика письма (ручной батч по очереди, к LLM — по одной карточке за шаг).

Параметры ниже передаются в harvest и перекрывают одноимённые переменные из .env. Один «проход» — поиск по ключам (см. логику ниже), сбор ссылок до лимита сессии, парсинг карточек, оценка LLM.

Одной кнопкой «Остановить поиск» создаётся флаг остановки для живого процесса и при необходимости дописывается завершение в лог (как раньше «Сбросить зависшее состояние»).

Настройки

        
      

      

        Подробный лог harvest (Debug) — JSONL в data/harvest-debug.log
        (остановки, process.exit, ошибки; в записи env секреты маскируются). На запуск
        передаётся HH_HARVEST_DEBUG=1. Если переменная на сервере не задана, состояние
